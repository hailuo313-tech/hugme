必须串行（不可两人同时改同一处）
  • 同一文件例如 .github/workflows/*.yml、同一段 llm_orchestrator.py：先合并一方 PR → 另一方 rebase 后再改。
  • 进入 main 的合并：同一时间只推进一条合并线（避免互相覆盖）。
  • 生产 git pull / 重启：与下一波开发 push 错开窗口，由 Codex 统一发「维护窗口」。

依赖顺序（短）：产品确认「vulnerability」与后端字段映射 → Cursor 接口与鉴权进 main → Devin 接真数据 → Codex 部署与冒烟。
