当前线上主路径：MTProto auto_reply.py 直处理（不经 Redis inbound_queue）、Bot Webhook、Open API 三路最终都调用 generate_reply。话术匹配：仅在 app_download_conversion 的 reply 钩子检索 script_templates；archive_worker 在消息发送后归档 script_hit。视频通话：与聊天主路径并行，仅 MTProto 入站触发；见下方策略表。
视频通话 · call_broadcast（生产已开 CALL_BROADCAST_ENABLED=1）：

与 asset_keyword（发图/发视频文件）不同，本模块处理实时视频通话：Telethon + PyTgCalls 外呼或接听后播放预置视频。Bot / Open API 不触发。

触发方式行为
用户发消息关键词命中 video call / facetime / cam2cam / 视频通话 等 → 入队 call_broadcast_jobs → worker 外呼播视频；不阻塞当轮 generate_reply
测试码 8866消息精确匹配 8866 → 入队后立即 spawn worker tick
用户主动来电第 1 次：自动接听，播 play_sequence=1 视频；第 2 次起（INBOUND_MANUAL_AFTER=1）：转坐席审核 admin/video-broadcast
坐席手动外呼后台 POST /api/v1/call-broadcast/admin/manual-calls 向指定用户发起通话
通话后POST_CALL_NURTURE_ENABLED=1 时可衔接 click_only 培育；每账号 MAX_CONCURRENT=1
三方推广 · click_only 转化策略（2026-06 线上）：

核心 KPI = CTR（click_users / link_exposed_users）。download/register 三方不可见，APP_DOWNLOAD_FUNNEL_MODE=click_only 下不参与决策。

入站与编排（channel / orchestrator）

三路实际上线入站共用 generate_reply；H5 仅传输层。编排含意图、孤独感、记忆、app_download 话术钩子；无入站安全门与出站一致性检查。

安全红线配置（redlines）

服务器 config/safety_filter_redlines.json 与 app/config/safety_filter_redlines.json 当前红线均为 0 条；页面不再展示旧的 safety 规则列表。

分级与接管（level / policy / operator）

calcUserLevel 写入 S/A/B/C/D；POL-01 七条件可建 handoff。S/A 入站不自动挂起坐席。

转化培育（conversion / nurture / worker）

click_only：漏斗止于点击，CTR 为核心 KPI。高意图 first_push/direct_cta/after_warmup/operator/asset skip LLM 直出；培育仅未点击用户；已点击后 worker +2min clicked_post_followup_2m，跟进后发完即收口。

视频通话（call_broadcast）

仅 MTProto：用户消息关键词或测试码 8866 入队外呼；用户主动来电第 1 次自动接听，第 2 次起坐席审核。PyTgCalls 播放 video_broadcast_assets（play_sequence 1/2/3）。管理端 admin/video-broadcast。
