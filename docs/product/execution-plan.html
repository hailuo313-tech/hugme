// 用户分级体系概览

S 级
专家坐席精聊

T1 国家 + 年龄 ≥ 25 + 高额付费记录。最高优先级，坐席实时接管。

A 级
专家坐席精聊

T1 国家 + 年龄 ≥ 25 + 暂未付费。重点转化目标，价值引导策略。

B 级
AI + 人工辅助

T2 国家或低客单付费记录。AI 主导，坐席辅助观察。

C 级
100% AI 托管

T2 国家 + 无付费记录。全自动运行，零坐席成本。

D 级
画像探测阶段

画像字段残缺，触发专属话术自动补全信息，待升级。

03

AI 处理核心链路

话术底料库 · 意图识别 · AI 拟人化包装 · 智能延迟投递

第 3–7 周
▾

📚话术底料库建设

  • 整理话术类目：问候 / 情感维系 / 增值权益引导 / 拒绝处理 / 付费催化
  • 每类话术录入 ≥ 10 条底料，存入 script_templates 表
  • 为每条底料打关键词标签（视频/相册/VIP/礼包…），支持向量化检索
  • 底料支持按 platform / user_level / persona 过滤检索
📄 scripts_seed.sql

🎯意图识别与话术匹配

  • 定义意图分类体系：内容请求 / 情感互动 / 投诉 / 付费咨询 / 闲聊
  • 关键词规则 + 向量嵌入双路召回，返回最相关话术底料
  • 输出结构：{ intent, script_template, confidence }
  • confidence < 0.6 时降级为通用闲聊话术，避免答非所问
📄 intent_classifier.ts

🤖AI 拟人化包装模块

  • 为每个人设配置专属系统 Prompt（性格 / 说话风格 / 语气 / 禁用词列表）
  • 输入组合：话术底料 + 最近 5 轮上下文 + 人设 Prompt → 调用大模型
  • 对输出做安全过滤：屏蔽真实身份暴露、价格硬承诺等高风险词汇
  • 包装结果存入 Redis 待发队列，附带目标 send_at 时间戳
📄 persona_prompts/ 📄 safety_filter.ts

⏱智能延迟投递引擎

  • 基于回复字数计算打字模拟延迟：delay = chars × 0.08s + 3s（可配置）
  • B/C/D 级：倒计时结束后异步自动投递，不占坐席资源
  • S/A 级：挂起自动投递，WebSocket 推送坐席弹窗等待接管操作
  • 坐席超时未响应（默认 120s）→ 自动执行兜底投递，记录日志
📄 delivery_queue.ts

📦数据归档：premium_chat_logs 写入

  • S/A 级用户每条消息（上行 + 下行）同步克隆写入 premium_chat_logs
  • 记录字段：user_id / operator_id / direction / message_text / captured_at
  • 归档操作为异步写入，不阻塞主链路响应
  • 提供管理端查询 API，支持按用户、按时段、按坐席检索精聊记录
📄 archive_service.ts ✅ 精聊记录 100% 可查
